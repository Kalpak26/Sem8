# Simple Parser for Conditional Statements

This project implements a basic **Lexical Analyzer** (using Lex/Flex) and a **Syntax Analyzer** (using Yacc/Bison) to validate a specific `IF-THEN` statement structure.

---

## 🛠️ How it Works

The execution is split into two main phases:

### 1. Lexical Analysis (`scanner.l`)
The scanner reads the input stream character by character and groups them into **tokens**. 
* **Keywords:** Recognizes `IF` and `THEN`.
* **Operators:** Identifies Relational Operators (`ROP`) like `<, >, ==` and Assignment/Arithmetic Operators (`AOP`) like `=`, `+`, etc.
* **Identifiers & Numbers:** Identifies variables (`ID`) and numeric constants (`NUM`).
* **Whitespace:** Ignored.

### 2. Syntax Analysis (`parser.y`)
The parser receives tokens from the scanner and checks if they follow the defined Context-Free Grammar (CFG):
* **Grammar Rule:** `IF <Condition> THEN <Statement>`
* It validates whether the comparison and the subsequent assignment/operation are syntactically correct.
* If the input matches the rules, it prints a success message; otherwise, `yyerror` is triggered.

---

## 🚀 Execution Commands

To run this program, ensure you have `flex`, `bison` (or `yacc`), and `gcc` installed on your system.

```bash
# 1. Generate the parser header and C file from Yacc
yacc -d parser.y

# 2. Generate the lexical analyzer C file from Lex
lex scanner.l

# 3. Compile the generated C files together
gcc lex.yy.c y.tab.c -o parser

# 4. Run the executable
./parser
```

---

## 📝 Example Usage

Once the program is running, you can test it with the following inputs:

**Valid Input:**
```text
IF a < b THEN x = 10
```
*Output: Rule triggered: ID AOP NUM. Input is valid according to the grammar.*

**Invalid Input:**
```text
IF a < THEN x = 10
```
*Output: Grammar Error: syntax error*

## 📂 File Description

| File | Description |
| :--- | :--- |
| `scanner.l` | The Lex specification file defining token patterns. |
| `parser.y` | The Yacc grammar file defining the structure and logic. |
| `y.tab.h` | Header file generated by Yacc containing token definitions (used by Lex). |
| `y.tab.c` | The C source code for the parser generated by Yacc. |
| `lex.yy.c` | The C source code for the lexer generated by Lex. |
