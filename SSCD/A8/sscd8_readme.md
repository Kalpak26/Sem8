# Lex and Yacc Calculator

This project implements a simple arithmetic calculator using **Lex** (Lexical Analyzer) and **Yacc** (Yet Another Compiler Compiler). It can evaluate basic mathematical expressions involving addition, subtraction, multiplication, division, and parentheses.

## How the Two Files are Connected

Lex and Yacc work together in a pipeline to process and evaluate the input:

1. **Lex (`calc.l`)** acts as the **Scanner**. It reads the raw input characters, groups them into meaningful chunks called **tokens** (like `NUMBER` or `+`), and sends them to Yacc.
2. **Yacc (`calc.y`)** acts as the **Parser**. It receives tokens from Lex by repeatedly calling the `yylex()` function. It then checks these tokens against grammatical rules to understand the structure of the expression (e.g., `NUMBER + NUMBER`) and evaluates it.

**The Bridge:** The two tools communicate primarily through the `yylex()` function and a special shared variable called `yylval`. When Lex finds a number, it stores its actual integer value in `yylval` so Yacc can use it for calculations. They also share a header file (`calc.tab.h`) generated by Yacc, which contains the token definitions that Lex uses.

## File Breakdown & Rules

### 1. The Lexical Analyzer (`calc.l`)
This file breaks the input text into recognizable tokens.
* `[0-9]+`: This regular expression matches one or more digits. When found, `atoi(yytext)` converts the text string into an integer, stores it in `yylval`, and tells Yacc it found a `NUMBER`.
* `"+"`, `"-"`, `"*"`, `"/"`, `"("`, `")"`: These rules match mathematical operators and parentheses, returning the characters themselves as tokens to Yacc.
* `[ \\t] ;`: This rule ignores spaces and tabs so they don't interfere with parsing.
* `\\n`: Returns a newline token to Yacc, which Yacc uses as a signal to print the final result of the line.

### 2. The Parser (`calc.y`)
This file defines the arithmetic grammar and how to evaluate it.
* **Precedence Rules (`%left`)**: These tell Yacc how to resolve ambiguities (Shift-Reduce conflicts).
  * `%left '+' '-'` and `%left '*' '/'` ensure that multiplication and division are evaluated before addition and subtraction, and that operators evaluate from left to right.
* **Grammar Rules**:
  * `expr '+' expr { $$ = $1 + $3; }`: If Yacc sees an expression, a plus sign, and another expression, it adds their values (`$1` and `$3`) and stores the result in `$$` (the parent node).
  * `line: expr '\\n'`: When a complete expression is followed by an "Enter" key (newline), it prints the calculated result.
* **`yyerror()`**: If the user enters something that doesn't fit the grammar (like `+ * 5`), this function handles the syntax error gracefully by printing "Invalid Expression".

## How to Compile and Execute

To run this calculator, you will need `flex` (or `lex`), `bison` (or `yacc`), and a C compiler like `gcc` installed on your system.

### 1. Compile the files
Run the following commands in your terminal:
```bash
# Step 1: Run Yacc with the -d flag to create the parser C file and the header file (calc.tab.h)
bison -d calc.y   # (Use yacc -d calc.y if bison is not installed)

# Step 2: Run Lex to create the scanner C file (lex.yy.c)
flex calc.l       # (Use lex calc.l if flex is not installed)

# Step 3: Compile both C files together into an executable named 'calc'
gcc calc.tab.c lex.yy.c -o calc
```
